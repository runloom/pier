# Codex 指挥中心物料终态设计

## 目标和完成标准

- Codex 只贡献“账号与配额”“成本”两个职责清晰的物料，不建立包含全部数据的总览卡。
- 两个物料按内容区宽度在窄、中、宽三档自动调整密度，不横向溢出，也不靠放大默认尺寸制造留白。
- 物料壳的标题栏保持紧凑；动作能放下时直接显示，空间不足时才进入“更多”。
- 额度使用稳定的数据蓝，成本使用稳定的分析青色；警告和危险色只表达真实风险。
- 设置页和物料复用配额、成本图表与格式化逻辑，账号和数据契约保持不变。

## 当前结构为什么不足

旧实现把动作全部放进“更多”，标题栏又通过分割线和过大留白抢占内容空间。Codex 卡片内部使用大面积弱化背景和固定尺寸布局，导致窄卡溢出、宽卡空洞；额度与成本还借用了主题主色或成功色，使数据系列与操作、状态语义混在一起。插件样式若依赖宿主构建期扫描，也无法保证受管理外部插件升级后的样式完整性。

## 所有权划分

| 层 | 负责内容 |
| --- | --- |
| 宿主物料壳 | 卡片结构、拖拽抓手、标题、动作排序、真实宽度测量、溢出菜单、异步失败反馈 |
| 插件注册协议 | 仅声明命令型动作；不允许插件自绘标题栏、嵌套菜单或任意节点 |
| Codex 插件 main | 账号切换、在线配额抓取、本机会话日志成本聚合 |
| Codex renderer 数据源 | 共享快照、分域刷新、可见性生命周期与重复请求合并 |
| Codex 物料 | 信息选择、容器查询响应、账号切换和数据呈现 |
| Codex 设置页 | 当前账号、成本、其他账号三个管理区块 |
| 产品主题 | 数据色和状态色的唯一具体颜色所有者 |
| 测试 | 动作溢出、配额排序、三档响应、深浅主题、交互反馈和样式治理 |

## 物料结构

### 账号与配额

- 账号行展示头像、邮箱、订阅、更新时间和条件性的可重置次数。
- 账号切换保留在账号行，使用独立的 `DropdownMenu`；它不是标题栏命令动作。
- 主配额按周期从短到长，其他主窗口随后；模型专属配额经分隔后放在末尾。
- 窄档只显示核心账号信息和最紧张主配额；中档显示两个主配额；宽档显示全部主配额，并在空间允许时显示模型专属配额。
- 标题栏提供刷新配额动作。

### 成本

- 窄档展示今日成本和 7 日趋势。
- 中档展示今日、31 日成本、31 日 tokens 和 31 日趋势。
- 宽档增加最近有效数据日 tokens，并在数据不完整时显示带说明的状态徽标。
- 图表统一使用 `ChartContainer`、`BarChart` 和 `ChartTooltip`；成本柱使用 `data-cost`，无估价数据使用弱化色。
- 标题栏提供刷新成本动作。

## 标题栏动作流

```text
插件动作 + 宿主动作
        │
        ├─ 按优先级稳定排序
        │
ResizeObserver 测量 Header、按钮和间距
        │
        ├─ 可用空间足够 → 直接图标按钮 + Tooltip
        └─ 空间不足     → 低优先级动作进入“更多”
```

菜单打开期间冻结分配，关闭后再同步测量，避免焦点目标在交互中移动。所有 Promise 动作由宿主提供等待态；未处理异常走宿主详情弹窗，成功不自动叠加 toast。

## 数据与样式流

```text
Codex CLI / OAuth ──> 插件账号与配额服务 ──> RPC 快照 ──> 账号与配额物料 / 设置页
本机会话日志 ──────> 插件成本聚合服务 ────> RPC 快照 ──> 成本物料 / 设置页
globals 数据令牌 ──> @pier/ui 语义映射 ──> 插件独立 Tailwind 构建 ──> 注入样式
```

插件构建通过 `@reference "@pier/ui/tailwind-theme.css"` 使用公开的语义映射，只扫描插件 renderer 源码并生成实际使用的无前缀 utility。生成规则放在 `@scope ([data-pier-codex-scope])` 中，只作用于插件贡献根节点和显式标记的 Portal，不参与宿主设置页等其他界面的级联。插件不反向读取宿主源码，也不重复输出主题定义、preflight、全局 reset 或宿主 Dockview 工具类。产品颜色值仍由宿主 `globals.css` 唯一维护，宿主不通过 `@source` 扫描 Codex 插件源码。

## 禁止的反模式

- 不允许插件自绘整个标题栏。
- 不允许动作在直接区和“更多”中重复出现。
- 不把所有动作默认塞进“更多”。
- 不硬编码配额名称、周期、Spark 是否存在或窗口数量。
- 不使用主题 `primary`、成功绿或业务文件中的具体颜色表达普通额度与成本。
- 不用物料 `size` 推算像素布局；内容响应只使用容器查询。
- 不让 renderer 扫描日志、保存凭据或重新定义 main 数据契约。
- 不把插件生成的无前缀 utility 注入全局作用域；插件 Portal 必须显式进入插件样式作用域。

## 验收矩阵

| 需求 | 证据 |
| --- | --- |
| 终态为两个物料 | `plugin.json` 只声明账号与配额、成本，旧组合或第三物料不存在 |
| 标题栏紧凑 | `MissionControlWidgetCard` 无分割线，完整使用 `CardHeader/CardTitle/CardAction/CardContent` |
| 动作按空间显示 | 宽度算法单测和 Electron 宽/窄卡交互测试 |
| 插件样式独立 | 插件构建产物包含 22rem/34rem 查询且无 preflight、Dockview 样式 |
| 数据色正确 | 颜色治理测试与深浅主题计算样式验证 |
| 动态配额 | 0／1／多主窗口、未知周期和模型专属窗口排序测试 |
| 响应无破版 | 2×3、4×3、6×4 与最大尺寸 Electron 截图 |
| 设置页一致 | 当前账号、成本、其他账号宽窄窗口组件与 Electron 测试 |
| 结构闭环 | 类型检查、插件打包、治理测试、组件测试和目标 E2E 全部通过 |
